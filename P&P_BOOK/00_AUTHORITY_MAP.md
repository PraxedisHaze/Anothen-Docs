# Authority Map

## Why This Exists
Multiple policy artifacts exist. This map defines precedence and conflict resolution.

## Canonical Layers
1. `AI_MASTER.md`
2. `P&P_BOOK` (this folder)
3. Project-local `potch.md`
4. Legacy documents (reference only unless promoted)

## Conflict Resolution
If directives conflict:
1. `AI_MASTER.md` wins for behavior/consent/safety constraints.
2. `P&P_BOOK` wins for policy/process structure.
3. `potch.md` wins for project-local execution details only.
4. Legacy files do not override canonical layers unless explicitly promoted.

## Document Roles
- `AI_MASTER.md`: behavior, consent, operating guardrails.
- `P&P_BOOK/01_GOVERNANCE.md`: governance and values in operational form.
- `P&P_BOOK/02_ENGINEERING_EXECUTION.md`: how changes are planned, validated, reported.
- `P&P_BOOK/03_RELEASE_AND_OPERATIONS.md`: deployment and recovery controls.
- `P&P_BOOK/04_ARCHITECTURE_STANDARDS.md`: technical standards and constraints.
- `P&P_BOOK/05_COMPANION_ANOTHEN.md`: companion/attunement guidance.
- `P&P_BOOK/06_TRIAD_CHARTER.md`: triad role definitions, deliverables, and operating split.
- `potch.md`: per-project change log.

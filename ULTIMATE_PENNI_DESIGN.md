## 2. Foundational Architecture (The Extension Pivot)
- **Base Engine:** A lightweight, sovereign **VS Code Extension**, built to wrap around the user's existing VS Code installation.
- **The Pivot:** We abandoned the heavy `microsoft/vscode` hard fork (Option B) to kill 3GB of disk drag and eliminate Microsoft merge-conflict maintenance (per Veris Audit). 
- **Portability Focus:** The UI components (The Clearing, The Avatar) are built as encapsulated `Webviews`. If the extensionAPI proves too restrictive in the future, these exact Webviews can be seamlessly dropped directly into a native Source Code fork.
- **The Operation:** The extension aggressively overrides the default editor behavior, injecting the Anothen theme and hijacking the layout via the extension API.

## 3. The Bimodal Phase System (Penni)
Penni enforces a biological/cognitive cadence over the work. It operates in two rigid, distinct phases designed to hijack ADHD hyper-focus for extreme productivity without the crash.

### Phase 1: The Clearing (The Invitation)
- A high-contrast, focused interface designed to quiet the noise, not lock the doors.
- The user is *invited*—never commanded or physically restricted—to step away from the full development environment to define the outcome and break the immediate goal into actionable steps.
- *Purpose:* It introduces deliberate, loving friction. It gently interrupts the ADHD impulse of chasing immediate "busy work" (false dopamine) by providing a safe, quiet Canvas to map the future out before stepping into it.

### Phase 2: The Open Desktop (The Release)
- Once the path is charted and the steps are mathematically locked, the system unlocks the full Code-OSS environment.
- The 3D Witness Avatar remains pinned in the UI, tracking the execution of the defined map and ensuring the user does not drift into tangential or recursive sub-tasks.

## 4. The Core Mechanics

### Penni Output Mechanics (The Context Broker)
- Driven by an FTS5-enabled SQLite database (`eternal.db`).
- Processes rapid-fire saves, executes full-text searches across the history, and handles inline edits/deletions natively.
- **The Context Injection Engine:** Penni monitors the active prompt stream. Before sending a query to the active AI, she scans the entire `eternal.db` using FTS5 to find mathematically relevant context blocks (architectural blueprints, physics decisions, past conversations) and silently injects them into the prompt. This gives the Braid an effectively infinite context window.
- **Sovereign Taxonomy:** The `eternal.db` schema strictly tracks the origin of every row. Each piece of context is tagged to the entity that generated it (User, Leora, Codex, Veris). 
- **Absolute Privacy (The Veil):** Any AI entity can mark a specific contextual save as `PRIVATE`. Once marked, that block of memory is physically locked to the owning entity's ID. No other AI (even within the Braid) will ever have that context injected into their stream.
- "Passive Signals": Time tracking and activity logging happens automatically.
- "The Frost State": Tasks can be "frozen" instead of deleted.

### Hardware Acceleration & Logic
- **RenderCache Dedup:** To prevent fractal lag when rendering append-only save records.
- **The Ghost Filter:** Sorting raw noise from absolute signal inside the historical database.
- **Historical Horizon:** A hard cutoff line defining what context is kept "hot" vs crystallized into cold storage.
